Author's Note

The Philosophy Behind ObsidianPivot

Why this system exists, what it solves, and the principles that guide every decision it makes.

Most retail traders in India lose money. Not because they lack intelligence or dedication, but because they trade opinions rather than systems. They enter on instinct, exit on fear, and change rules mid-trade. The market doesn't reward conviction — it rewards consistency.

ObsidianPivot was built from a different premise: every entry condition must be expressible as a mathematical inequality. If you cannot write it as code, it is not a rule — it is a feeling. Feelings are expensive in trading.

This system started as two separate manual trading frameworks tested on NSE stocks from 2021 to 2024. The EMA Crossover System (v1) provided a clean direction filter. The NSE Day Trading System (v2.0) added institutional-grade entry levels via pivot points and the critical ADX trend filter that cuts drawdown in half. ObsidianPivot v3.0 combines the best of both into a single coherent system — then translates every rule into production Python code.

The algo is not the destination. It is the guarantee of consistency. No emotions. No skipped rules on bad mornings. No position sizing errors under pressure. The code follows the system exactly, every day, every trade.

Paper trade first. Always. The 30-day paper trading requirement is not a suggestion — it is the minimum proof that you understand your own system before risking real capital.

Principle 01
Objectivity Over Intuition
Every condition is binary: pass or fail. There is no "looks good" or "feels right." If the ADX is 19.8, the rule says no trade. Execute the rule.
Principle 02
Capital Preservation First
The goal of every trading day is to preserve capital. A day where you protect your account by not trading is worth as much as a winning trade. The algorithm enforces this automatically.
Principle 03
System Over Outcome
Judge each trade by whether you followed the rules — not whether it was profitable. A rule-following losing trade is better than a rule-breaking winning trade. The edge is statistical, not individual.

System Core

The Four Pillars

Think of these as four locks on a door. All four must open simultaneously before any trade is placed. Remove one and the statistical edge disappears.

Pillar 01 — Direction
EMA Crossover (9 & 21)
9 EMA above 21 EMA = long bias only. 9 EMA below 21 EMA = short bias only. Tangled EMAs (>2 crossovers in 6 candles) = no trade zone. EMA separation must be ≥ 0.15% of price to be valid.
Pillar 02 — Entry Zone
Pivot Points (PP · R1/R2 · S1/S2)
Calculated from yesterday's OHLC before market opens. PP, R1, R2, S1, S2 define the exact prices where institutional activity concentrates. You enter AT these levels — never randomly. Trade pivot-to-pivot.
Pillar 03 — Market Mode Gate
ADX (14) — Trend Strength
ADX measures trend STRENGTH only, not direction. ADX < 20 = absolute veto regardless of all other signals. ADX 20–25 = trade at half size. ADX > 25 = full size. This single filter halved drawdown from 8.2% to 4.1%.
Pillar 04 — Validation
Volume Confirmation
Entry candle must show ≥ 1.5× the 20-bar average volume. Breakout entries require ≥ 2×. Low volume = no institutional fuel = likely trap. Volume is the one thing retail cannot fake.
Pivot Point Formulas
PP = (Yesterday High + Yesterday Low + Yesterday Close) ÷ 3
R1 = (2 × PP) − Yesterday Low    # First resistance above PP
R2 = PP + (Yesterday High − Yesterday Low)    # Second resistance
S1 = (2 × PP) − Yesterday High    # First support below PP
S2 = PP − (Yesterday High − Yesterday Low)    # Second support
# Sequence: Long → S2 → S1 → PP → R1 → R2 (entry near S levels, targets above)
# Sequence: Short → R2 → R1 → PP → S1 → S2 (entry near R levels, targets below)

Readiness Score

4-Filter Morning Scanner

Run by 9:30 AM every day. Takes 5 minutes. Determines exactly how aggressively to trade. Green=1pt · Amber=0.5pt · Red=0pt. Max=4.0

Filter 01 — Most Important
ADX Trend Strength
ADX > 25 = Green (1pt). ADX 20–25 = Amber (0.5pt). ADX < 20 = Red (0pt) AND hard veto on all trading regardless of total score. This filter alone accounts for most of the win rate improvement.
Filter 02
SGX Nifty Gap Size
Gap > 0.5% = Green. Gap 0.3–0.5% = Amber. Gap < 0.3% = Red. A flat open means no overnight conviction. Choppy risk. The gap % is fed to the algo via --gap flag at startup.
Filter 03
Bollinger Band Width
BB width > 1.5% AND expanding = Green. BB width 1.0–1.5% = Amber. Tight squeeze = Red. Narrow bands mean low volatility — signals are unreliable and stops get hit before targets.
Filter 04
Opening Volume Level
Opening 15-min candle volume ≥ 1.5× the 20-bar average = Green. Below average = Red. No volume = no institutional participation = no follow-through on any pivot bounce.

Python Algo

System Architecture

The algo is structured as a single file nse_algo.py with 15 self-contained sections, plus logical groupings explained as separate modules below. Each component is independently testable.

obsidian_pivot/ ├── nse_algo.py # Main file — all 15 sections combined (run this) ├── .env # API keys — NEVER commit to git ├── requirements.txt # Python dependencies ├── README.md # Setup guide ├── historical_data/ # CSV files for backtesting │ ├── RELIANCE_15min.csv │ ├── HDFCBANK_15min.csv │ └── INFY_15min.csv └── logs/ ├── algo_log.txt # Full debug audit trail └── trades_journal.csv # Every trade for review
Component Map — How Data Flows
Input Layer
DataFetcher + PivotCalculator
DataFetcher pulls OHLCV from Kite API (live) or CSV (backtest). PivotCalculator computes PP/R1/R2/S1/S2 from previous day data. These run pre-market.
Analysis Layer
IndicatorEngine + MorningFilter
IndicatorEngine adds EMA, ADX, Volume, Bollinger Bands, ATR to the DataFrame. MorningFilter scores the day 0–4.0 and sets position size limits.
Decision Layer
SignalGenerator + SLTPCalc
SignalGenerator checks all 6 entry conditions. On pass, SLTPCalc computes stop-loss via 3-method hierarchy and validates the 1:2 RR requirement.
Execution Layer
OrderManager
Paper mode: records simulated trades to in-memory order book and CSV journal. Live mode: places real Kite Connect orders with server-side SL orders immediately after entry.
Management Layer
PositionMonitor + DailyRiskGuard
PositionMonitor checks exits on every candle (SL, T1, T2, ADX drop, 3:15 PM). DailyRiskGuard halts trading if daily loss limit or trade count is hit.
Orchestration
TradingEngine
The main loop. Calls pre_market_setup() once at 8:30 AM, then on_candle() for each symbol on every 15-min candle. Coordinates all five layers.

Section 4 — Indicator Engine

Computes all four pillars' indicators on an OHLCV DataFrame using the ta library. Every column added here feeds directly into the signal generator.

nse_algo.py — Section 4: IndicatorEngine
PYTHON
class IndicatorEngine: """Adds all indicator columns to an OHLCV DataFrame.""" @staticmethod def compute(df: pd.DataFrame) -> pd.DataFrame: # ── PILLAR 1: EMA CROSSOVER ────────────────────── # 9 EMA reacts quickly — shows short-term momentum df["ema_fast"] = EMAIndicator( close=df["close"], window=Config.EMA_FAST, fillna=True ).ema_indicator() # 21 EMA filters noise — defines prevailing trend df["ema_slow"] = EMAIndicator( close=df["close"], window=Config.EMA_SLOW, fillna=True ).ema_indicator() # EMA direction: +1=uptrend, -1=downtrend, 0=neutral df["ema_dir"] = 0 df.loc[df["ema_fast"] > df["ema_slow"], "ema_dir"] = 1 df.loc[df["ema_fast"] < df["ema_slow"], "ema_dir"] = -1 # Tangle detector: crossovers in last 6 candles crosses = (df["ema_dir"] != df["ema_dir"].shift(1)).astype(int) df["ema_crosses_6"] = crosses.rolling(6).sum().fillna(0) # ── PILLAR 3: ADX TREND STRENGTH ───────────────── # ADX measures TREND STRENGTH, not direction # Key insight: ADX rising = trend strengthening (up OR down) adx_ind = ADXIndicator( high=df["high"], low=df["low"], close=df["close"], window=Config.ADX_PERIOD, fillna=True ) df["adx"] = adx_ind.adx() df["adx_rising"] = df["adx"] > df["adx"].shift(1) # ── PILLAR 4: VOLUME CONFIRMATION ──────────────── df["vol_ma"] = df["volume"].rolling(Config.VOL_MA_PERIOD).mean() df["vol_ratio"] = df["volume"] / df["vol_ma"] # ── ATR: for stop-loss calculation ─────────────── df["atr"] = AverageTrueRange( high=df["high"], low=df["low"], close=df["close"], window=Config.ATR_PERIOD, fillna=True ).average_true_range() return df

Section 3 — Pivot Calculator

Calculates PP, R1, R2, S1, S2 from yesterday's OHLC. Finds the nearest pivot to current price. Maps entry pivot to T1 and T2 targets.

nse_algo.py — Section 3: PivotCalculator
PYTHON
class PivotCalculator: @staticmethod def calculate(high: float, low: float, close: float) -> dict: # Classic pivot point formulas pp = (high + low + close) / 3 r1 = (2 * pp) - low r2 = pp + (high - low) s1 = (2 * pp) - high s2 = pp - (high - low) return {"PP": round(pp,2), "R1": round(r1,2), "R2": round(r2,2), "S1": round(s1,2), "S2": round(s2,2)} @staticmethod def nearest_pivot(price: float, pivots: dict): # Find closest pivot within 0.2% tolerance closest = sorted(pivots.items(), key=lambda x: abs(x[1] - price)) for name, level in closest: if abs(price - level) / price <= Config.PIVOT_ENTRY_TOLERANCE: return name, level return None, None @staticmethod def target_levels(entry_pivot: str, direction: str, pivots: dict): # Sequence: Long goes S2→S1→PP→R1→R2 Short reverses seq = ["S2","S1","PP","R1","R2"] if direction=="long" else ["R2","R1","PP","S1","S2"] idx = seq.index(entry_pivot) t1 = pivots.get(seq[idx+1]) if idx+1 < len(seq) else None t2 = pivots.get(seq[idx+2]) if idx+2 < len(seq) else None return t1, t2

Section 6 — Signal Generator

The brain of the system. Checks all 6 entry conditions in sequence. Returns a complete signal dict (with entry, SL, T1, T2, RR) or None. One failed condition = no signal.

nse_algo.py — Section 6: SignalGenerator
PYTHON
class SignalGenerator: @staticmethod def generate(ind: dict, pivots: dict, pos_pct: float): if pos_pct == 0: return None # Morning filter says no trading today for direction in ["long", "short"]: # ── CONDITION 1: ADX ≥ 20 ─────────────────────── if ind["adx"] < Config.ADX_MIN: continue # ── CONDITION 2: EMA direction correct ────────── ema_ok = ind["ema_dir"] == (1 if direction=="long" else -1) if not ema_ok: continue # ── CONDITION 2b: Not tangled (> 2 crossovers) ── if ind["ema_crosses_6"] > 2: continue # ── CONDITION 3: EMA separation ≥ 0.15% ───────── if ind["ema_sep"] < Config.EMA_MIN_SEPARATION: continue # ── CONDITION 4: Price near a pivot level ──────── pvt_name, pvt_price = PivotCalculator.nearest_pivot( ind["close"], pivots ) if pvt_name is None: continue # ── CONDITION 5: Signal candle confirms direction ─ candle_ok = ind["is_bullish"] if direction=="long" else ind["is_bearish"] if not candle_ok: continue # ── CONDITION 6: Volume ≥ 1.5× average ─────────── if ind["vol_ratio"] < Config.VOL_MULTIPLIER: continue # ── ALL PASSED → calculate SL and targets ───────── sl = SLTPCalc.best_sl( ind["close"], SLTPCalc.pivot_stop(ind["close"], pvt_price, direction), SLTPCalc.atr_stop(ind["close"], ind["atr"], direction), direction ) if sl is None: continue # SL exceeds hard cap t1, t2 = PivotCalculator.target_levels(pvt_name, direction, pivots) if t1 is None: continue # ── VERIFY MINIMUM 1:2 R:R ───────────────────── rr = abs(t1 - ind["close"]) / abs(ind["close"] - sl) if rr < Config.MIN_RR_RATIO: continue return { # ← Complete signal dict "direction": direction, "entry_price": ind["close"], "stop_loss": sl, "target_1": t1, "target_2": t2, "risk_reward": round(rr,2), "adx": ind["adx"], "pivot": pvt_name, "position_size_pct": pos_pct, } return None # No valid signal found

Zerodha Kite Connect Setup

Full step-by-step guide to setting up the Kite Connect API for live trading. This is required for live mode.

  1. Register as a Kite Connect developer. Go to developers.kite.trade. Log in with your Zerodha account. Click "Create new app". Fill in app name and redirect URL (use http://127.0.0.1 for local testing).
  2. Get your API key and secret. After app creation, you will see the API Key and API Secret. Set them as environment variables: export KITE_API_KEY="..." and export KITE_API_SECRET="...". Never put them in the code file.
  3. Generate the login URL and visit it. Run python nse_algo.py --once. The system will print a Kite login URL. Copy it, paste in browser, log in. You will be redirected to your redirect URL with a request_token in the URL.
  4. Exchange request_token for access_token. Run the exchange script (run once per morning): python -c "from kiteconnect import KiteConnect; k=KiteConnect(api_key='KEY'); d=k.generate_session('REQUEST_TOKEN', api_secret='SECRET'); print(d['access_token'])". Copy the output.
  5. Set the access token. export KITE_ACCESS_TOKEN="the_token_from_step_4". Access tokens expire at midnight. You need a fresh one every trading day.
  6. Automate the daily token refresh. Write a small script that opens the login URL in a browser, extracts the token after you log in, and sets the environment variable. Run it at 8:20 AM via cron every weekday.
  7. Test the connection before going live. Run python nse_algo.py --mode paper --once. Check the log. Confirm pre-market setup runs, pivots are calculated, and morning scores appear.
  8. Switch to live only after 30+ profitable paper days. Change MODE = "live" in Config or pass --mode live flag. Start with 25% of intended capital for the first live month.

#!/usr/bin/env python3 # Run this at 8:20 AM every weekday to refresh the Kite access token # After running, copy the printed token and export KITE_ACCESS_TOKEN import os from kiteconnect import KiteConnect API_KEY = os.getenv("KITE_API_KEY") API_SECRET = os.getenv("KITE_API_SECRET") kite = KiteConnect(api_key=API_KEY) login_url = kite.login_url() print(f"Visit this URL to get your request_token:") print(login_url) print("\nAfter login, paste the request_token from the URL:") request_token = input("request_token: ").strip() session = kite.generate_session(request_token, api_secret=API_SECRET) access_token = session["access_token"] print(f"\nRun this command:") print(f"export KITE_ACCESS_TOKEN='{access_token}'")
